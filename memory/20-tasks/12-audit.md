## Найденные и исправленные баги

### 🔴 CRITICAL: Утечка hostname в `make_msgid()`

| Поле | Значение |
|---|---|
| **Приоритет** | 🔴 CRITICAL |
| **Категория** | Безопасность / Приватность |
| **Локация** | `core/sender.py` — формирование Message-ID |
| **Описание** | `make_msgid()` без параметра `domain` использует `socket.getfqdn()`, раскрывая реальный hostname машины отправителя в заголовке `Message-ID` |
| **Исправление** | Передача домена SMTP-аккаунта: `make_msgid(domain=smtp_account.domain)` |
| **Статус** | ✅ Исправлено |

> [!CAUTION]
> Утечка hostname — серьёзная проблема: спам-фильтры и получатели могут идентифицировать отправителя.

### 🔴 CRITICAL: Утечка SMTP-соединения при ошибке `sendmail()`

| Поле | Значение |
|---|---|
| **Приоритет** | 🔴 CRITICAL |
| **Категория** | Утечка ресурсов |
| **Локация** | `core/sender.py` — блок отправки |
| **Описание** | Если `sendmail()` выбрасывает исключение, `conn.quit()` не вызывается → соединение остаётся открытым → утечка сокетов |
| **Исправление** | Обёртка в `try/finally` с гарантированным `conn.quit()` |
| **Статус** | ✅ Исправлено |

```python
# До:
conn = smtp_connect(account)
conn.sendmail(from_addr, to_addrs, msg.as_string())
conn.quit()

# После:
conn = smtp_connect(account)
try:
    conn.sendmail(from_addr, to_addrs, msg.as_string())
finally:
    conn.quit()
```

### 🟡 SPAM: X-Mailer заголовок как fingerprint массовой рассылки

| Поле | Значение |
|---|---|
| **Приоритет** | 🟡 SPAM (средний) |
| **Категория** | Антиспам / Доставляемость |
| **Локация** | `core/sender.py` — формирование MIME |
| **Описание** | Статический `X-Mailer: SMTP MAILER` позволял спам-фильтрам идентифицировать и группировать все письма как массовую рассылку |
| **Исправление** | Рандомизация заголовка из пула реальных почтовых клиентов (`Thunderbird`, `Apple Mail`, `Outlook`, `The Bat!` и др.) |
| **Статус** | ✅ Исправлено |

### 🟡 SPAM: CSV-экспорт без полей `had_cc` / `had_bcc`

| Поле | Значение |
|---|---|
| **Приоритет** | 🟡 SPAM (низкий) |
| **Категория** | Полнота данных |
| **Локация** | `core/stats.py` — CSV-экспорт |
| **Описание** | В CSV-отчёте отсутствовали поля `had_cc` и `had_bcc`, что мешало анализу доставляемости по типу рассылки |
| **Исправление** | Добавлены поля `had_cc` и `had_bcc` в CSV-экспорт |
| **Статус** | ✅ Исправлено |

### 🟢 MINOR: Нет backoff при 4xx soft-bounce

| Поле | Значение |
|---|---|
| **Приоритет** | 🟢 MINOR |
| **Категория** | Надёжность |
| **Локация** | `core/sender.py` — обработка ошибок |
| **Описание** | При получении 4xx ответа (temporary failure) повтор выполнялся немедленно без задержки → сервер мог заблокировать отправителя |
| **Исправление** | Добавлен retry-sleep с экспоненциальной задержкой перед повтором |
| **Статус** | ✅ Исправлено |
