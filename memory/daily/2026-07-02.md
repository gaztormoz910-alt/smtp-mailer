## Ключевые выводы

### 1. `formataddr` для доверия
Использование `email.utils.formataddr` с RFC 2047 кодированием — обязательно для кириллических имён отправителя. Без этого письма попадают в спам или отображаются с «кракозябрами». См. [[30-decisions]].

### 2. BCC только в envelope
BCC-адреса должны быть **исключительно** в envelope (`RCPT TO:`), а **не** в MIME-заголовках. Добавление BCC в заголовки полностью нарушает концепцию скрытой копии. Это одна из самых частых ошибок в реализации массовых рассылок. См. [[30-decisions]].

### 3. Рандомизация X-Mailer
Статический заголовок `X-Mailer` — простой и эффективный маркер для спам-фильтров. Рандомизация из пула реальных клиентов (`Thunderbird`, `Apple Mail`, `Outlook`, `The Bat!`) значительно снижает spam score. См. [[40-errors]] — BUG-003.

### 4. Cleanup соединений
`try/finally` для `conn.quit()` — критически важно при массовой рассылке. Тысячи утёкших сокетов могут привести к исчерпанию ресурсов ОС и полной остановке рассылки. См. [[40-errors]] — BUG-002.
